# AgentLayer v0.1.1

AgentLayer v0.1.1 is an adoption patch for the first public release. It keeps the product scope
unchanged and focuses on making the project easier to try, understand, and share.

## Live Demo

Live read-only demo:

https://agentlayer-readonly-demo.vercel.app

Hosted demo uses the AcmeFlow fixture. Run the CLI locally for real sites.

## Try A Real Site

AgentLayer is still documented for repo-local use:

```bash
git clone https://github.com/Qqqq5910/agentlayer.git
cd agentlayer
pnpm install
pnpm build
```

Generate artifacts and run the doctor command for a public site:

```bash
pnpm agentlayer generate https://example.com --out ./agentlayer-output --max-pages 20
pnpm agentlayer doctor https://example.com --max-pages 20
```

For the local AcmeFlow fixture:

```bash
pnpm dev:example
pnpm agentlayer generate http://localhost:3001 --out ./agentlayer-output --max-pages 20 --allow-local
pnpm agentlayer doctor http://localhost:3001 --max-pages 20 --allow-local
```

## What Changed

- README and Chinese README now put the live read-only demo first.
- CLI quickstart examples now use the same positional URL syntax as the actual CLI.
- Feedback docs explain how to share real scan results with URL, command, score, wrong facts or
  actions, confusing recommendations, and artifacts users would publish.
- Launch-copy docs make the project easier to share without inventing new product claims.
- GitHub metadata docs capture the recommended description, website, topics, social preview, and
  pinned feedback issue.

## Scope

- No arbitrary hosted scanner is enabled.
- The hosted demo remains read-only and fixture-based.
- No scanner behavior, scoring behavior, or generated artifact behavior changes are included.

# AgentLayer v0.1.0

AgentLayer v0.1.0 is the first public release for a deterministic
agent-operability toolkit. It helps teams check whether a public website can be
read, trusted, and operated by AI agents without relying on an LLM judge by
default.

## Highlights

- Scan public websites within same-host, page-limit, timeout, and robots.txt
  bounds.
- Extract sourced facts, action paths, form operability hints, and
  standards-ready draft artifacts.
- Run deterministic B2B SaaS task checks for pricing, docs, security,
  integrations, support, and demo/contact paths.
- Generate `llms.txt`, `llms-full.txt`, Markdown snapshots, `.well-known`
  drafts, WebMCP suggestions, JSON reports, and `report.html`.
- Try the local Next.js demo at `http://localhost:3000/demo` with the AcmeFlow
  fixture at `http://localhost:3001`.

## Install And Try

AgentLayer is currently documented for repo-local use:

```bash
pnpm install
pnpm build
pnpm dev:example
pnpm agentlayer generate http://localhost:3001 --out ./agentlayer-output --max-pages 20 --allow-local
pnpm agentlayer doctor http://localhost:3001 --max-pages 20 --allow-local
```

For a public site:

```bash
pnpm agentlayer generate https://example.com --out ./agentlayer-output --max-pages 20
pnpm agentlayer test https://example.com --tasks ./examples/tasks/b2b-saas.default.json --out ./agentlayer-report.json
```

## Notes

- Generated MCP, WebMCP, API Catalog, and Agent Skills files are conservative
  drafts, not compliance claims.
- The scanner does not authenticate, submit forms, crawl private areas, or
  perform destructive actions.
- npm package execution is planned, but the recommended v0.1.0 path is the
  repo-local `pnpm agentlayer` CLI.
